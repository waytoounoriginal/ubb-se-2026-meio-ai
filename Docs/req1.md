Assignment 1
You have formed a team and talked with the client; this is happening. The client wants to see details set in stone before you get the go ahead. So, the team decides to sit down and try to figure out how to do this before jumping into code.
•
Write down all requirements in a document that you agreed upon. They need to be clear, complete, verifiable, consistent and feasible. Each requirement must have a clear owner. For cross team requirements you need the other team’s name also. What you agree to here is what you will implement for the next assignment. The next assignment (in two weeks from now) will be to fully implement all requirements until the lab 3.
•
Create a use case diagram based on your requirements.
•
Create a database diagram for your application.
•
Your design must use an MVVM pattern for the GUI. The actual GUI’s look and feel will be done during your second seminar. You must have more than one view in the GUI. You must store data long term.
•
Create a class diagram depicting all your application, so everything from the user interface to the data storage including domain, utils etc. Generated diagrams are worth 0 points. If your second seminar comes after your second lab then don’t include your GUI in the design that you present and add it afterwards.
•
Create tasks in a project management software for the implementation of your application and any other task related to completing assignment 2. All team members must have access to the software. Each task represents at most 30 minutes of work, has a relevant name and the description is detailed enough that anybody could implement it if they wanted to. So, if you have 2 tasks that means that all of them can be done in an hour. This will likely generate many tasks, so if you have less than 40 then probably you are not done.
•
Define for yourselves who is responsible for creating tasks, and how they are assigned. For example, let’s say everybody is responsible to add any work that they deem to the board, and everybody can pick any task, then non-negligible work would mean adding or doing multiple tasks. If for example the team lead does all tasks and assigns them, then negligible work is not adding the tasks and assigning them or not doing the assigned task or doing so very late. But in this case a student who did not get any tasks or receiving few even after asking for it, is not considered to have done negligible work.
•
Do not write any code, this assignment is all about design and planning.
